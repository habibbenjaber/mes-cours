Partie Théorique Détaillée

1. Qu'est-ce que les props en React ?

Les props (abréviation de "properties") sont le mécanisme de React pour passer des données d'un composant parent à un composant enfant. Elles sont :

  • Immuables (read-only) : Le composant enfant ne peut pas modifier ses props
  • Unidirectionnelles : Les données vont toujours du parent vers l'enfant
  • Similaires aux attributs HTML : Syntaxe similaire aux attributs des balises HTML

Composant Parent

Passe les données vers le bas

↓

Composant Enfant

Reçoit les données en lecture seule

Props (Propriétés)

  • Données passées au composant
  • Immuables (read-only)
  • Communication parent → enfant
  • Similaires aux paramètres de fonction

State (État)

  • Données internes au composant
  • Mutable (peut être modifié)
  • Géré localement avec useState
  • Déclenche des re-rendus

2. Syntaxe de base des props

Les props sont passées comme attributs dans le JSX et reçues comme paramètre dans le composant.

// Composant enfant qui reçoit des props
function Bienvenue(props) {
  return <h1>Bonjour, {props.nom} !</h1>;
}

// Utilisation avec les props
function App() {
  return (
    <div>
      <!-- Passage des props comme attributs -->
      <Bienvenue nom="Marie" />
      <Bienvenue nom="Pierre" />
    </div>
  );
}

3. Destructuration des props

La destructuration permet d'extraire directement les props dans les paramètres.

// Sans destructuration
function Utilisateur(props) {
  return (
    <div>
      <h2>{props.nom}</h2>
      <p>Âge : {props.age}</p>
    </div>
  );
}

// Avec destructuration
function Utilisateur({ nom, age }) {
  return (
    <div>
      <h2>{nom}</h2>
      <p>Âge : {age}</p>
    </div>
  );
}

4. Types de données qu'on peut passer en props

  • Strings : nom="Ali"
  • Numbers : age={25}
  • Booleans : isAdmin={true}
  • Arrays : items={["a", "b", "c"]}
  • Objects : user={{name: "Salem", age: 30}}
  • Functions : onClick={handleClick}
  • JSX/Components : children={"..."}

5. La prop spéciale "children"

La prop children permet de passer du contenu JSX entre les balises d'un composant.

// Composant avec children
function Card({ children, title }) {
  return (
    <div className="card">
      {title && <h3>{title}</h3>}
      {children}
    </div>
  );
}

// Utilisation
<Card title="Mon Titre">
  <p>Ce contenu est passé via children</p>
  <button>Cliquez ici</button>
</Card>

6. Props par défaut avec defaultProps

Vous pouvez définir des valeurs par défaut pour les props.

// Avec la syntaxe moderne
function Bouton({ text = "Cliquez-moi", color = "blue" }) {
  return (
    <button style={{ backgroundColor: color }}>
      {text}
    </button>
  );
}

// Utilisation avec valeurs par défaut
<Bouton /> // Utilise les valeurs par défaut
<Bouton text="Valider" color="green" />

7. Passage de fonctions en props

Les fonctions peuvent être passées en props pour permettre la communication enfant → parent.

// Composant parent
function Parent() {
  const handleChildClick = (message) => {
    console.log("Message de l'enfant:", message);
  };

  return (
    <Enfant onButtonClick={handleChildClick} />
  );
}

// Composant enfant
function Enfant({ onButtonClick }) {
  return (
    <button onClick={() => onButtonClick("Bonjour Parent!")}>
      Cliquez pour envoyer un message
    </button>
  );
}

⚠️ Règles importantes des props :

  • Les props sont en lecture seule - Ne jamais les modifier directement
  • Utilisez la destructuration pour un code plus lisible
  • Nommez les props clairement - Utilisez des noms descriptifs
  • Props vs State - Les props viennent de l'extérieur, le state est interne
