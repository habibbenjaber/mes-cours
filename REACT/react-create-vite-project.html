1. Qu'est-ce que Vite ?

Vite (qui signifie "rapide" en français) est un outil de build moderne qui offre une expérience de développement extrêmement rapide pour les projets web.

Serveur de développement rapide

fournit de riches améliorations de fonctionnalités par rapport aux modules ES natifs

HMR Ultra-Rapide

Hot Module Replacement qui met à jour les modules sans recharger la page

Build Optimisé

qui regroupe votre code avec Rollup , préconfiguré pour générer des ressources statiques hautement optimisées pour la production.

Riches de Fonctionnalités

Support TypeScript, JSX, CSS et plus encore

🛠️

4. Création d'un Projet React avec Vite

Installation et démarrage

npm create vite@latest mon-projet-react -- --template react
cd mon-projet-react
npm install
npm run dev

Structure du projet

mon-projet-react/
├── public/
│ └── vite.svg
├── src/
│ ├── components/
│ ├── App.jsx
│ ├── main.jsx
│ └── index.css
├── index.html
├── package.json
└── vite.config.js

💡 Points Importants

Vite utilise des extensions .jsx pour les fichiers React et configure automatiquement tout ce dont vous avez besoin pour développer avec React.

📁

3. Structure d'un Projet Vite - Relation entre Fichiers

Dans un projet React avec Vite, trois fichiers principaux travaillent ensemble pour afficher votre application :

📄
index.html
Point d'entrée principal de l'application
Contient la div racine où React sera rendu
⚛️
main.jsx
Point d'entrée JavaScript
Monte l'application React dans le DOM
🧩
App.jsx
Composant racine de l'application
Contient la structure principale de l'UI

Relation entre les fichiers :

index.html → importe main.jsx
main.jsx → importe et rend App.jsx
App.jsx → importe d'autres composants

Contenu de index.html :

<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8" />
  <title>Mon App React</title>
</head>
<body>
  <!-- Point de montage pour React -->
  <div id="root"></div>
  <!-- Vite injecte main.jsx ici -->
  <script type="module" src="/src/main.jsx"></script>
</body>
</html>

Contenu de main.jsx :

import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'
import './index.css'

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
)

Contenu de App.jsx :

import Header from './components/Header.jsx'
import Main from './components/Main.jsx'
import Footer from './components/Footer.jsx'

function App() {
  return (
    <div className="app">
      <Header />
      <Main />
      <Footer />
    </div>
  )
}

export default App;
🔄

4. Import et Export des Composants

Pour utiliser un composant dans un autre composant, vous devez l'exporter depuis son fichier et l'importer dans le fichier où vous souhaitez l'utiliser.

📤 Exporter un composant :

// Dans Header.jsx
function Header() {
  return <header><h1>Mon Site</h1></header>
}

// Export par défaut
export default Header;

// Ou export nommé
export { Header };

📥 Importer un composant :

// Dans App.jsx
// Import par défaut
import Header from './components/Header.jsx';

// Import nommé
import { Header } from './components/Header.jsx';

function App() {
  return (
    <div>
      <!-- Utilisation du composant importé -->
      <Header />
    </div>
  )
}

✅ Bonne Pratique

Utilisez l'export par défaut pour les composants principaux et l'export nommé pour les composants utilitaires ou les exports multiples depuis un même fichier.

🧩

5. Les Composants React - Les Bases

Un composant React est une fonction JavaScript qui retourne du JSX (JavaScript XML) pour décrire ce qui doit être affiché à l'écran.

Composant Fonctionnel Simple

function Bienvenue() {
  return (
    <div>
      <h1>Bonjour le monde !</h1>
      <p>Ceci est mon premier composant React.</p>
    </div>
  );
}

Composant avec Syntaxe Arrow Function

const Titre = () => {
  return <h2>Mon Application React</h2>;
};

🧱 Bâtir avec des Composants

Les applications React sont construites en assemblant des composants réutilisables, comme des blocs de construction.

🎯 Un Rôle Unique

Chaque composant a une responsabilité spécifique dans l'interface utilisateur.

♻️ Réutilisabilité

Un même composant peut être utilisé plusieurs fois dans différentes parties de l'application.

🔗

6. Composition de Composants

Les composants peuvent utiliser d'autres composants pour créer des interfaces complexes.

Exemple de Composition avec Import/Export

// Composant En-tête (Header.jsx)
function Header() {
  return <header><h1>Mon Site</h1></header>;
}
export default Header;

// Composant Principal (Main.jsx)
function Main() {
  return (
    <main>
      <p>Contenu principal de l'application</p>
    </main>
  );
}
export default Main;

// Composant Application (App.jsx)
import Header from './Header.jsx';
import Main from './Main.jsx';

function App() {
  return (
    <div>
      <Header />
      <Main />
    </div>
  );
}
export default App;

✅ Bonne Pratique

Divisez votre interface en petits composants réutilisables. Chaque composant devrait avoir une responsabilité unique et claire.

🎓 Exercice Final - Application avec Import/Export

Scénario : Créez une application avec plusieurs composants organisés dans différents fichiers.

Structure à créer :

  • Header.jsx - En-tête de l'application
  • Navigation.jsx - Barre de navigation
  • MainContent.jsx - Contenu principal
  • Footer.jsx - Pied de page
  • App.jsx - Composant principal qui importe tous les autres
// Header.jsx
function Header() {
  return <header><h1>Mon Application</h1></header>;
}
export default Header;

// Navigation.jsx
function Navigation() {
  return (
    <nav>
      <a href="#home">Accueil</a>
      <a href="#about">À propos</a>
    </nav>
  );
}
export default Navigation;

// App.jsx
import Header from './Header.jsx';
import Navigation from './Navigation.jsx';
import MainContent from './MainContent.jsx';
import Footer from './Footer.jsx';

function App() {
  return (
    <div>
      <Header />
      <Navigation />
      <MainContent />
      <Footer />
    </div>
  );
}
export default App;
